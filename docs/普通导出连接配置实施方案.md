# 普通导出连接配置实施方案

## 目标

新增普通连接配置导出能力，导出 `dbx-config` 格式的 JSON 文件，支持内网 HTTP 或 IP 地址场景；现有加密导出能力继续保留。

## 功能设计

### 导出入口

在连接配置导出操作中提供两种明确的导出方式：

- 加密导出：沿用现有密码短语和 `dbx-encrypted` 文件格式。
- 普通导出：直接导出 `dbx-config` 文件，不依赖 Web Crypto、HTTPS 或 localhost。

普通导出执行前必须弹出风险确认，提示文件包含数据库密码、SSH 凭据及其他连接敏感信息，确认后才生成文件。

### 普通导出数据

普通导出复用现有连接配置组装逻辑，固定生成以下结构：

```json
{
  "format": "dbx-config",
  "version": 1,
  "connections": [],
  "layout": {
    "groups": [],
    "order": []
  },
  "tunnelProfiles": []
}
```

数据路径固定为：连接配置状态 -> 导出负载 -> `dbx-config` JSON 文件 -> 文件下载。

不新增字段兼容、字段回退或双格式写入逻辑；普通导出只使用当前连接配置、侧边栏布局和隧道配置字段。

### 导入处理

沿用现有导入入口，根据文件格式分流：

- `dbx-encrypted`：继续要求密码短语并执行解密。
- `dbx-config`：直接解析并导入，不弹出密码短语对话框。

普通导入继续应用连接分组和隧道配置，保持现有导入后的关联关系及覆盖规则。

## 代码调整

### 前端

- 在 `connectionStore` 中抽取统一的连接配置导出负载构造逻辑。
- 增加普通导出方法，序列化为 `dbx-config` 后下载 `dbx-connections.json`。
- 保留现有加密导出方法，仅调整导出方式选择和成功提示调用。
- 新增普通导出确认对话框，明确展示敏感信息风险。
- 更新中英文及已支持语言的导出相关文案，区分普通导出和加密导出。

### 后端

不增加普通导出的后端接口。普通导出只在浏览器端完成 JSON 序列化和文件下载，不访问数据库、不写入服务端存储。

### 类型与格式

- 复用现有 `PlainConfigPayload` 类型，并补齐布局及隧道配置字段定义。
- `format` 固定为 `dbx-config`，`version` 固定为 `1`。
- 加密导出格式、加密参数和既有加密导入协议保持不变。

## 安全约束

- 普通导出必须经过用户主动选择和二次确认，不能作为默认导出方式静默执行。
- 风险提示必须明确说明导出文件包含连接凭据。
- 普通导出文件不得写入日志、服务端存储或调试信息。
- 不修改现有加密导出，不将普通导出作为加密不可用时的自动兜底路径。

## 交付边界

本方案只新增普通连接配置导出，不改变数据库导出、SQL 文件导出、第三方工具导入和桌面端文件选择行为。
